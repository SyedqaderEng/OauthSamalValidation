Frequently Asked Questions

How long do OAuth apps and SAML environments stay active?
All OAuth applications and SAML environments automatically expire after 30 days. Three days before expiration, you'll receive an email with XML backups of all your configurations and metadata. This ensures you have a record while keeping the platform clean and performant.
Can I use MockAuth for production applications?
MockAuth is designed for development and testing purposes only. It's perfect for building and validating your OAuth/SAML integrations, running automated tests, and debugging authentication flows. For production use, please use a production-grade identity provider like Okta, Auth0, or Azure AD.
Are the decoder tools completely free?
Yes! Our JWT Decoder, SAML Decoder, and Base64 tools are completely free and serverless. They run entirely in your browser, so your data never leaves your machine. No sign-up required for these tools.
How do I import metadata from my Identity Provider?
When creating a SAML Service Provider, you can upload your IdP's metadata XML file directly. MockAuth will automatically parse it and configure the SSO URL, SLO URL, entity ID, and certificate. Alternatively, you can manually enter these values if you prefer.
Can I automate OAuth app creation via API?
Absolutely! MockAuth provides a REST API for programmatic access. Create API keys in your settings, then use them to create, manage, and delete OAuth apps and SAML environments. This is perfect for CI/CD pipelines and automated testing.
What OAuth 2.0 grant types are supported?
MockAuth supports all major OAuth 2.0 grant types: Authorization Code, PKCE (Proof Key for Code Exchange), Client Credentials, and Refresh Token. You can enable multiple grant types per application and customize token lifetimes.
Is my data secure on MockAuth?
Yes. All connections use TLS encryption, passwords are hashed using bcrypt, and API keys are stored securely. Remember, MockAuth is for testing only - never use production secrets or sensitive data. All test environments are automatically deleted after 30 days.
Can I test both SP-initiated and IdP-initiated SAML flows?
Yes! MockAuth supports both Service Provider-initiated and Identity Provider-initiated SAML flows. When you create a SAML environment, you'll get endpoints for both flow types and can test each one with our built-in testing tools.
