Welcome to MockAuth

MockAuth is a powerful testing platform for OAuth 2.0 and SAML 2.0 integrations. Whether you're building a new authentication system or testing an existing one, MockAuth provides all the tools you need to develop and validate your implementation.

With MockAuth, you can create mock OAuth providers and SAML identity providers in seconds, test various authentication flows, decode and analyze tokens, and ensure your integration works correctly before going to production.

OAuth 2.0 Example

Authorization Code Flow

The Authorization Code flow is the most commonly used OAuth 2.0 flow for web applications. Here's how to implement it with MockAuth:

Step 1: Create an OAuth Application

Navigate to your dashboard and click "Create OAuth App". Configure your redirect URIs and select the Authorization Code grant type.

Step 2: Redirect User to Authorization URL

const authUrl = `https://mockauth.dev/oauth/authorize? client_id=${CLIENT_ID}& response_type=code& redirect_uri=${REDIRECT_URI}& scope=read:profile write:data& state=${RANDOM_STATE}`; window.location.href = authUrl;

Step 3: Exchange Authorization Code for Token

const response = await fetch('https://mockauth.dev/oauth/token', { method: 'POST', headers: { 'Content-Type': 'application/x-www-form-urlencoded' }, body: new URLSearchParams({ grant_type: 'authorization_code', code: authorizationCode, redirect_uri: REDIRECT_URI, client_id: CLIENT_ID, client_secret: CLIENT_SECRET }) }); const { access_token, refresh_token } = await response.json();

SAML 2.0 Example

SP-Initiated SSO Flow

Service Provider-initiated SSO is the most common SAML flow. Here's how to test it with MockAuth:

Step 1: Create a SAML IdP

Create a SAML Identity Provider in MockAuth and configure your attribute mappings. Download the IdP metadata XML.

Step 2: Configure Your SP

Import the MockAuth IdP metadata into your Service Provider application. Configure your SP's Entity ID and ACS URL.

Step 3: Initiate SSO

When a user attempts to access a protected resource, your SP will redirect them to the MockAuth IdP SSO URL with a SAML AuthnRequest.

Step 4: Process SAML Response

After authentication, MockAuth will POST a SAML Response to your ACS URL. Parse and validate the assertion to complete the login.

API Reference

Authentication

All API requests must include your API key in the Authorization header:

Authorization: Bearer YOUR_API_KEY

Common Endpoints

  • GET /v1/oauth/apps - List all OAuth applications
  • POST /v1/oauth/apps - Create a new OAuth application
  • GET /v1/oauth/apps/:id - Get OAuth application details
  • DELETE /v1/oauth/apps/:id - Delete an OAuth application
  • GET /v1/saml/environments - List all SAML environments
  • POST /v1/saml/environments - Create a new SAML environment

For complete API documentation, visit our API Explorer.
